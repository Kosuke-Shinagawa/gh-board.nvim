# 要件定義: gh-board.nvim

> 作成日: 2026-06-13
> 形式: USDM（Universal Structured Data Model）

---

## 要求: R-01 Kanban ボード表示

**要求**: `:GhBoard` コマンドで GitHub Projects v2 の Kanban ボードをフロートウィンドウに表示できる
**理由**: ブラウザを開かずに nvim 内で Kanban の全体像を把握することが、このプラグインの起点となる操作であるため

### 仕様

| 仕様ID | 仕様 | 実装状況 |
|--------|------|---------|
| R-01-S01 | `:GhBoard` コマンドでフロートウィンドウを開く | ⬜ 未実装 |
| R-01-S02 | 設定済みプロジェクトのステータスフィールドをカラムとして横並びに表示する | ⬜ 未実装 |
| R-01-S03 | 各カラム内にカードタイトルを縦に列挙する | ⬜ 未実装 |
| R-01-S04 | カラムの表示順は GitHub API から取得した順序をそのまま使用する | ⬜ 未実装 |
| R-01-S05 | データ取得中はローディングインジケーターを表示する | ⬜ 未実装 |
| R-01-S06 | `r` キーでボードを最新データに手動リフレッシュできる | ⬜ 未実装 |
| R-01-S07 | `q` または `<Esc>` でボードを閉じる | ⬜ 未実装 |

---

## 要求: R-02 カード詳細表示

**要求**: ボード上のカードを選択してタイトル・本文・メタ情報の詳細を閲覧できる
**理由**: カードの内容確認はブラウザで行う最も頻繁な操作であり、nvim 内で完結させることでコンテキストスイッチを排除するため

### 仕様

| 仕様ID | 仕様 | 実装状況 |
|--------|------|---------|
| R-02-S01 | ボード上でカーソル位置のカードに `<Enter>` を押すと詳細ポップアップを表示する | ⬜ 未実装 |
| R-02-S02 | 詳細ウィンドウにタイトル・本文・担当者・ラベル・作成日・更新日を表示する | ⬜ 未実装 |
| R-02-S03 | カードが Issue または PR にリンクされている場合、その番号・タイトル・状態を表示する | ⬜ 未実装 |
| R-02-S04 | `q` または `<Esc>` で詳細ウィンドウを閉じてボードに戻る | ⬜ 未実装 |

---

## 要求: R-03 カードのステータス変更

**要求**: ボード上のカードを別のカラム（ステータス）へ移動し GitHub Projects に同期できる
**理由**: Kanban 運用の中心操作であるステータス更新をブラウザなしで行うため

### 仕様

| 仕様ID | 仕様 | 実装状況 |
|--------|------|---------|
| R-03-S01 | ボード上でカードを選択し `m` キーで移動先カラムの選択メニューを表示する | ⬜ 未実装 |
| R-03-S02 | 移動先を選択すると `updateProjectV2ItemFieldValue` ミューテーションを実行する | ⬜ 未実装 |
| R-03-S03 | API 成功後にボードを再描画する | ⬜ 未実装 |
| R-03-S04 | API エラー時はエラーメッセージを `vim.notify` で通知し、状態をロールバックする | ⬜ 未実装 |

---

## 要求: R-04 カードの新規作成

**要求**: nvim 内からフォームを開いて新しいカードをプロジェクトに追加できる
**理由**: タスクの追記のたびにブラウザを開く必要をなくすため

### 仕様

| 仕様ID | 仕様 | 実装状況 |
|--------|------|---------|
| R-04-S01 | ボード上で `n` キーで新規作成フォームを開く | ⬜ 未実装 |
| R-04-S02 | フォームにタイトル（必須）と本文（任意）を入力できる | ⬜ 未実装 |
| R-04-S03 | タイトルが空の状態で確定しようとした場合はバリデーションエラーを表示する | ⬜ 未実装 |
| R-04-S04 | 確定すると `addProjectV2DraftIssue` ミューテーションを実行し Draft Issue として追加する | ⬜ 未実装 |
| R-04-S05 | 作成成功後にボードを再描画し、新しいカードを表示する | ⬜ 未実装 |
| R-04-S06 | `<Esc>` でフォームをキャンセルし、入力内容を破棄してボードに戻る | ⬜ 未実装 |

---

## 要求: R-05 カードの編集

**要求**: 既存カードのタイトル・本文を nvim 内で編集し GitHub Projects に同期できる
**理由**: カードの詳細修正をブラウザなしで行うため

### 仕様

| 仕様ID | 仕様 | 実装状況 |
|--------|------|---------|
| R-05-S01 | カード詳細ウィンドウで `e` キーを押すと編集フォームを開く | ⬜ 未実装 |
| R-05-S02 | 編集フォームに既存のタイトル・本文が初期値として入力されている | ⬜ 未実装 |
| R-05-S03 | Draft Issue カードの場合は `updateProjectV2DraftIssue` ミューテーションを実行する | ⬜ 未実装 |
| R-05-S04 | Issue/PR にリンクされたカードの場合は `updateIssue` ミューテーションを実行する | ⬜ 未実装 |
| R-05-S05 | 保存後に詳細ウィンドウを更新データで再描画する | ⬜ 未実装 |

---

## 要求: R-06 カードの削除

**要求**: カードをプロジェクトから削除し GitHub Projects に同期できる
**理由**: 不要になったカードの整理をブラウザなしで行うため

### 仕様

| 仕様ID | 仕様 | 実装状況 |
|--------|------|---------|
| R-06-S01 | ボードまたは詳細ウィンドウで `d` キーで削除確認ダイアログを表示する | ⬜ 未実装 |
| R-06-S02 | 確認ダイアログで `y` を入力すると `deleteProjectV2Item` ミューテーションを実行する | ⬜ 未実装 |
| R-06-S03 | 削除成功後にボードを再描画し、カードを除去する | ⬜ 未実装 |
| R-06-S04 | `n` または `<Esc>` でキャンセルし、何も変更しない | ⬜ 未実装 |

---

## 要求: R-07 認証

**要求**: 複数の認証方法をフォールバックチェーンで自動解決し、ユーザーが環境に応じた方法でトークンを提供できる
**理由**: gh CLI 利用者・環境変数派・直接設定派など利用環境が多様であり、すべてに対応することで設定の摩擦をなくすため

### 仕様

| 仕様ID | 仕様 | 実装状況 |
|--------|------|---------|
| R-07-S01 | `gh auth token` コマンドの出力からトークンを取得する（最優先） | ⬜ 未実装 |
| R-07-S02 | gh CLI が見つからない場合または失敗した場合は `$GITHUB_TOKEN` 環境変数をフォールバックとして使用する | ⬜ 未実装 |
| R-07-S03 | 環境変数もない場合は `setup({ token = "..." })` オプションをフォールバックとして使用する | ⬜ 未実装 |
| R-07-S04 | いずれでも取得できない場合は設定方法を案内するエラーメッセージを `vim.notify` で表示する | ⬜ 未実装 |
| R-07-S05 | `vim.fn.exepath("gh")` でパスを解決し、Windows / WSL 両環境に対応する | ⬜ 未実装 |

## 要求: NF-01 パフォーマンス

**要求**: API 呼び出し中に nvim の UI をブロックせず、描画を高速に完了する
**理由**: UI のブロックはエディタの操作性を損ない、プラグインの実用性を著しく低下させるため

### 仕様

| 仕様ID | 仕様 | 実装状況 |
|--------|------|---------|
| NF-01-S01 | すべての GitHub API 呼び出しは非同期（`vim.fn.jobstart` または `luv`）で実行し nvim の UI をブロックしない | ⬜ 未実装 |
| NF-01-S02 | Kanban 初期表示は API レスポンス受信後 200ms 以内に描画完了する | ⬜ 未実装 |

---

## 要求: NF-02 設定容易性

**要求**: lazy.nvim で 5 行以内の設定で動作すること
**理由**: 設定の手軽さがプラグイン採用の障壁を下げるため

### 仕様

| 仕様ID | 仕様 | 実装状況 |
|--------|------|---------|
| NF-02-S01 | `require("gh_board").setup({})` の呼び出しだけで最低限動作する（必須設定項目なし） | ⬜ 未実装 |
| NF-02-S02 | `:help gh-board` でキーマップ・設定オプション一覧を参照できる | ⬜ 未実装 |

## スコープ外

以下は v0.1.0 では実装しない。

| 項目 | 理由 |
|------|------|
| GitHub Projects v1（クラシック）のサポート | v2 と API が全く異なり、保守コストが高い |
| Issue / PR 自体の作成・マージ | Kanban カードの操作に集中する。Issue 操作は別プラグインの領域 |
| リアルタイムポーリング | API レート制限のリスクあり。手動リフレッシュ（`r`）で代替 |
| luarocks への公開 | Git 配布のみで十分（lazy.nvim が URL から直接インストール可能） |
| コメントの投稿・編集 | Nice to Have（v0.2.0 以降） |
| 複数プロジェクト切り替え UI | Nice to Have（v0.2.0 以降） |
| Telescope 統合 | Nice to Have（v0.2.0 以降） |
| Org プロジェクトサポート | Nice to Have（v0.2.0 以降） |
